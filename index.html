# Open Source 的定義 (1/2)

* 自由再散布
* 原始碼
* 衍生著作
* 原創作者程式原始碼的完整性
* 不得對任何人或團體有差別待遇
* 對程式在任何領域內的利用不得有差別待遇：可商業使用

---

# Open Source 的定義 (2/2)

* 散布授權條款
* 授權條款不得專屬於特定產品
* 授權條款不得限制其他軟體
* 授權條款必須技術（散佈媒介）中立

## [Bruce Perens](http://en.wikipedia.org/wiki/Bruce_Perens)

# 所以 Open Source 軟體是：

* 任何人可以自由地使用、改變和分享
* 多人共同開發
* 會宣告 licence 去保護其自由開放的權利

# Open Source 的要素 (1/3)

## 創新
* 解決某些特定的問題([visionmedia](https://github.com/visionmedia): [express](https://github.com/visionmedia/express), [mocha](https://github.com/visionmedia/mocha), [EJS](https://github.com/visionmedia/ejs))

## 文化
* 開放 – 什麼事情都可以改變，不過方案要好到讓大部份人同意（吵架？）
* 版主文化 – [Module Owner/Peer](https://wiki.mozilla.org/Modules/All#JS_Marionette_Harness_and_Tools)
* 自己擦屁股

---

# Open Source 的要素 (2/3)

## 溝通
* 公開分享資訊的平台 – [Wiki](https://wiki.mozilla.org/FirefoxOS/productivity)
* 公開討論事情的平台 – [Mailing List](https://groups.google.com/forum/#!forum/mozilla.dev.gaia)
* 公開即時通訊的平台 – [IRC](https://wiki.mozilla.org/IRC)

---

# Open Source 的要素 (3/3)

## 軟體工程
* Issue Tracker([Bugzilla](https://bugzilla.mozilla.org), [Issues](https://github.com/visionmedia/mocha/issues)) – 專案管理
* 多人協同工作([GitHub](https://github.com/mozilla-b2g/gaia)) – 版本管理
* [Code Review](https://github.com/mozilla-b2g/gaia/pull/19058) – 一致性、可維護的程式碼
* [Continuous Integration](https://travis-ci.org/mozilla-b2g/gaia) – build software at every change
